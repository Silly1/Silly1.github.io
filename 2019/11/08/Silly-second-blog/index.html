本篇文章将讲述我如何通过查找网络资料，一步步搭建博客的过程。

背景前提：

最开始是因为要学习Github以及GIT版本控制系统的使用，在B站学习视频 学会Git玩转Github的最后一节接触到Github Pages搭建网站的相关知识,但因为视频录制在16年，在某些步骤存在出入，并且现在Github Pages需要花钱了。

所以在坚守解决网络问题先不随便花钱的原则上，我带着奇心在B站上又看了一些基于Github搭建博客的视频，于是就了解到了Hexo+Github博客搭建模式。

在搭建过程中了解到还有别的博客搭建的方法，这里我不分析谁好谁坏，仅作技术记录，等后面学深了，再进行比较吧。

Hexo+Github博客搭建

安装环境：Windows 10(64位)

第一步：工具下载
1. Github网站基本使用+Git版本控制系统

2. Node.js软件(本节主要讲这部分)

软件下载：

官方网站 (我选的版本是12.13.0LTS)

选择目录进行安装

修改好安装目录后傻瓜式安装即可
安装完后可以通过cmd窗口输入node -v命令、npm -v命令分别查看node、npm版本号，若有版本号显示，则表示成功安装

注意：安装完Node.js后要重启电脑，不然会出现类似：bash:node command not found的错误

环境配置

我认为这一步很重要，你可以很好的控制如npm安装的全局模块所在路径以及缓存cache的路径。因为在默认情况下是安装到C盘的，这会在你不知不觉中占用到C盘空间。

该过程我参考了博客 周瑜周

可以去这上面看具体的截图过程，我下面仅做步骤记录，过程截图就不放上来啦！

  1. 在node.js安装的文件夹下新建两个文件【node_global】【node_cache】,在【node_global】下新建【node_modules】

  2. 创建完空文件夹后，打开cmd命令窗口（或在桌面下右键+GIT BASH HERE），在对话框中输入命令：

    npm config set prefix “D:\Code_QQcube\Nodejs\node_global”

    npm config set cache “D:\Code_QQcube\Nodejs\node_cache”

可以到C盘用户文件夹下找到你当前的用户名文件夹，打开名为.npmrc文件，里面将会记录着

prefix=D:\Code_QQcube\Nodejs\node_global
cache=D:\Code_QQcube\Nodejs\node_cache

  1. 设置环境变量：关闭cmd窗口，“我的电脑”-“右键”-“属性”-“高级系统设置”-“高级”-“环境变量”；进入环境变量对话框，在【系统变量】下新建【NODE_PATH】,输入【D:\Code_QQcube\Nodejs\node_global\node_modules】,将【用户变量】下的【Path】修改路径为【D:\Code_QQcube\Nodejs\node_global】

  2. 测试：配置完以上后，打开cmd窗口（或桌面下右键+GIT BASH HERE），输入npm install express -g 命令，安装express模块（-g是全局安装的意思）安装完成后，你可以看到expree的版本号。如果在该【D:\Code_QQcube\Nodejs\node_global\node_modules】路径下，可以找到express，则表示上面的配置成功。
